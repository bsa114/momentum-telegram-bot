# Momentum — объединённый бот (обычный режим + режим Инспектора)

Один Telegram-бот, один токен, два режима поведения. Переключение происходит
автоматически: как только боту приходит xlsx-файл, чат переходит в режим
Инспектора (заполнение чек-листа тайного гостя по аудио визита). Всё
остальное время бот работает как раньше — текст/голос через Dify, фото/PDF
через OCR.

## Что залить в существующий репозиторий (bsa114/momentum-telegram-bot)

Загрузить ВСЕ файлы, сохранив структуру (включая подпапку prompts/):

- `bot.py` — заменяет текущий bot.py
- `requirements.txt` — заменяет текущий requirements.txt
- `inspector_fill.py` — новый файл
- `inspector_transcription.py` — новый файл (транскрибация длинного аудио
  через DashScope fun-asr; переименован из transcription.py, чтобы не
  конфликтовать с именами внутри bot.py)
- `xlsx_structure.py` — новый файл
- `xlsx_writer.py` — новый файл
- `link_download.py` — новый файл (скачивание больших аудио по ссылке —
  Яндекс.Диск/Google Drive, в обход лимита Telegram в 20 МБ)
- `temp_file_server.py` — новый файл (временная раздача аудио из памяти)
- `prompts/inspector_agent_v2.md` — новый файл, обязательно в подпапке prompts/

## Новые переменные окружения на Render

К уже существующим (TELEGRAM_BOT_TOKEN, DASHSCOPE_API_KEY, DIFY_API_KEY)
нужно добавить:

- `OPENROUTER_API_KEY` — тот же ключ OpenRouter, что уже используется в Dify
  (модель заполнения анкеты идёт через anthropic/claude-sonnet-5 на OpenRouter,
  отдельный ключ Anthropic заводить не нужно)
- `PUBLIC_BASE_URL` — уже известный адрес сервиса,
  https://momentum-telegram-bot.onrender.com (без слеша на конце)

Больше ничего менять не нужно — .python-version, keep-alive через
cron-job.org и всё остальное остаётся как есть.

## Как это будет выглядеть в Telegram

Обычная работа — как раньше, без изменений.

Чтобы запустить заполнение анкеты:
1. Прислать боту пустую xlsx-анкету → бот подтверждает переход в режим
   Инспектора и просит аудио
2. Прислать аудио визита (файлом до 20 МБ, или ссылкой на Яндекс.Диск/
   Google Drive для более крупных записей)
3. Бот присылает расшифровку, затем либо сразу готовый xlsx, либо список
   уточняющих вопросов
4. Ответить одним сообщением вида "1-3, 2-1, 3-4" — бот досоставит и
   пришлёт финальный xlsx

Команда /reset в любой момент прерывает режим Инспектора и возвращает к
обычной работе бота.

## Важно — ещё не проверено вживую

Реальный вызов DashScope (fun-asr) и Claude через OpenRouter в связке пока
не тестировался на живых данных. Формат запроса к OpenRouter проверен
структурно (правильный endpoint, заголовки, JSON), но не с реальным ключом.
Первый прогон стоит сделать на коротком тестовом аудио и небольшой анкете.
